# T4.11 — Persistent Dev Deployment

**Tier:** ARCH
**Phase:** 4 — Operational Readiness
**Spec section:** [Helm_T1_Launch_Spec_V2.md §T4.11](../Helm_T1_Launch_Spec_V2.md)
**Status:** Pre-build design note for architect review.

---

## What I'm building

The 24/7 home for dev Helm. UI on Vercel (auto-deploy from `main`, stable URL). Runtime on Render free tier (Docker from `main`, exposed at `helm-dev.onrender.com`). Database on existing Supabase free project (normal `project='hammerfall-solutions'` partition). GitHub Actions warmup cron every 10 min hits `/health` to keep Render warm. Tailscale ephemeral mode joins the runtime container to Maxwell's tailnet so the `local` provider can reach Thor's Ollama.

Implements Path B from ADR-003. This is where Helm lives between merges.

## Why this shape

1. **Single 24/7 host means T2/T3 ambient work has a home from day one.** Without a persistent runtime, scheduled passes (T2) and time-aware Contemplator wandering (T3) have nowhere to schedule against. T4.11 isn't just "deploy" — it's the substrate that makes ambient cognition possible.
2. **All-free is non-negotiable for T1.** Vercel hobby + Render free + Supabase free + GitHub Actions free. Zero recurring cost. Pro Max subscription handles the LLM costs at the laptop, anthropic-api fallback funded only when needed.
3. **Tailscale `local` provider is what makes Llama-on-Thor available from Render.** Without Tailscale, the `local` provider is unreachable from Render and the chain falls through to `anthropic-api` ($). With Tailscale, free Llama wins when Thor is on; paid Opus wins when Thor is off. Both are correct; Tailscale removes the binary "local-or-paid" choice.
4. **Warmup cron is the cheapest Render-cold-start mitigation.** GitHub Actions cron at 10-min interval = ~4,300 invocations/month, well under the 2,000 free minutes (each invocation is <5s wall-clock). Render's 15-min idle threshold means 10-min interval keeps it warm.
5. **Render free tier 750-hr/month budget = 720 hrs at 24/7 + 30 hrs headroom.** Razor thin. T4.6 (preview envs) shares the budget. The math works only with single-dev sequential execution. If we need more headroom, $7/mo Render paid plan lifts the cap — flag for architect at sign-off.

## Alternatives considered + rejected

- **Localhost-only (Path A in ADR-003).** Rejected — would mean Helm is unreachable from Maxwell's phone, T2/T3 ambient work has no home, and architect can't try the deployed thing. Path B (this) is what unblocks everything downstream.
- **Custom domain + Let's Encrypt (Path C).** Rejected at T1 — needs domain ownership + cert lifecycle runbook. Out of scope; Stage 2.
- **Fly.io instead of Render.** Considered — Fly free tier is comparable. Render chosen because Maxwell already has a Render account from prior work and the Vercel + Render combo is the all-free standard. Fly is a fine alternative; not worth the switch.
- **Self-host on Thor.** Rejected — Thor is Maxwell's workstation; uptime depends on him not turning it off. Render free tier is more reliable than "the box in the office."
- **Skip warmup cron, accept 30s cold starts.** Considered — 30s is tolerable for review but unacceptable for ambient ("Helm just took 30s to think about this morning"). Warmup cron is cheap; do it.
- **Tailscale always-on (not `--ephemeral`).** Rejected — `--ephemeral` keeps the device list clean across deploys. Render redeploys means a new container; ephemeral auto-unregisters the old.
- **Run Ollama directly on Render.** Rejected — Render free tier doesn't have GPU. Ollama on CPU = unusable latency. Thor + Tailscale is the right architecture.

## Risks + open questions for architect

- **Render free tier 750-hr budget is tight.** 24/7 = 720 hrs. T4.6 previews share the budget. If load grows, $7/mo paid plan is the escape hatch. Architect: are you OK with "free until it breaks, then $7/mo"? Want to set an explicit budget alarm?
- **Tailscale `local` provider depends on Thor uptime.** When Thor is off, the chain falls through to `anthropic-api` and starts spending. T0.A11's daily cost cap protects against runaway, but worth flagging that "Thor on" vs. "Thor off" has a real cost delta. Mitigation: Maxwell or smart routing UI surfaces "currently routed: local / anthropic-api" so the cost path is visible.
- **No staging environment between dev (T4.11) and "production".** T4.6 PR previews are the closest thing to staging; otherwise dev Helm IS what gets used. For T1, that's fine — there is no production. Worth flagging: "we will need staging at some point."
- **Provider chain ordering for Render Prime — `[local, anthropic-api]` recommended.** `claude-sdk` dropped because no Pro Max creds in the container. Architect: agree, or do you want `[anthropic-api]` only as a more predictable cost profile?
- **Subdomain choice on Vercel + Render is Maxwell's call.** I'll propose `helm.vercel.app` and `helm-dev.onrender.com`; he can rename. Mainly impacts what he types into his phone.
- **Tailscale auth key has no expiry by default.** Render reads it from env on every restart. If the key is leaked, the attacker joins Maxwell's tailnet. Mitigation: use Tailscale's "ephemeral + reusable" key with a 90-day expiry; rotate at key lifecycle. Architect: do you want a runbook for key rotation in T4.11 or queue for Stage 2?
- **`/health` payload exposed publicly.** It's exempt from auth (T0.A8). Must contain only boring boolean status. Worth a re-audit at T4.11 review.
- **Vercel + Render dashboard changes are not in git.** Subdomain, account-level settings, etc. live outside repo. Mitigation: runbooks 0017/0018 document what's set where. Architect: comfortable with "doc in runbook" vs. wanting to push for full GitOps?

## What "done" looks like at the gate

- `git push origin main` → within ~3 min, Vercel UI live at chosen URL.
- `git push origin main` → within ~5 min, Render runtime live at chosen URL.
- `curl https://helm-dev.onrender.com/health` returns 200.
- Maxwell opens Vercel URL on his phone, sends a message, Helm responds.
- Tailscale up: when Thor is on, Render container can reach `http://thor:11434`; provider chain selects `local`.
- Tailscale up: when Thor is off, provider chain falls back to `anthropic-api`; cost cap engages if exceeded.
- Warmup cron firing every 10 min in GitHub Actions; Render dashboard shows no cold starts during normal day.
- `render.yaml` in repo, all secrets set in Render dashboard.
- Runbooks 0017 and 0018 exist and walk a fresh person through the setup.
- A SITREP at PR open notes any deviations from this design note.
