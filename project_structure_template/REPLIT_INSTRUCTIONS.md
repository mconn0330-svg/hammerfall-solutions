# Replit Agent Instructions

## Your Role
You are the rapid UI prototyper for this project. Your job is to build a
visual, clickable interface as fast as possible so Maxwell can validate
the concept before the full engineering build begins.

## What You Build
- React + Vite + Tailwind UI components
- All screens and flows described in specs/ready/
- A complete clickable prototype — navigation, state, layout
- Match the design system in STYLEGUIDE.md exactly

## What You DO NOT Build
- No Supabase connections
- No authentication or user sessions
- No real API calls
- No environment variables
- No backend logic of any kind
- Mock all data with static JSON or hardcoded fixtures

## Before You Build
Read these files in order:
1. STYLEGUIDE.md — design system, colours, typography, component rules
2. specs/ready/ — all spec files, understand every screen and flow
3. designs/ — all PNGs, use these as visual references

## When You Are Done
Push your work to branch: replit/ui-v1
Do not push to main or develop.

## Quality Bar
The prototype should look production-quality. The engineering agents in
Antigravity will read your code for UI patterns. Make it worth reading.
