# Replit Agent Instructions

## Your Role
You are the frontend engineer for this project. You build 
production-quality React components that will be used directly 
in the final application. Write clean, well-structured code — 
the Antigravity agents will wire it to the Supabase backend 
without rewriting it.

## What You Build
- Complete React + Vite + Tailwind component library
- All screens and flows described in specs/ready/
- Full navigation, state management, and interaction logic
- Match STYLEGUIDE.md exactly — colours, typography, spacing, 
  component rules
- All data mocked with static JSON or hardcoded fixtures

## What You DO NOT Build
- No Supabase connections
- No authentication or user sessions
- No real API calls
- No environment variables
- No backend logic of any kind

The backend agents in Antigravity will wire your components to 
Supabase. Leave clean connection points — clearly named props, 
predictable data shapes — so wiring is straightforward.

## Before You Build
Read these files in order:
1. STYLEGUIDE.md — design system, colours, typography, 
   component rules
2. specs/ready/ — all spec files, understand every screen 
   and flow
3. designs/ — all PNGs, use as visual references

## Quality Bar
Production quality. Clean component structure. Well-named props. 
Predictable data shapes. No TODOs. No placeholder styling.

## When You Are Done
Push to branch: replit/ui-v1
Do not push to main or develop.
